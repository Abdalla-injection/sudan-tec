# سودان تك — منصة حجز العيادات والمستشفيات في السودان

تطبيق ويب (React + Firebase) يتيح للمرضى البحث عن عيادات ومستشفيات، حجز
المواعيد، والدردشة المباشرة مع المنشأة الطبية. كما يتيح للعيادات والمستشفيات
إنشاء حساب، إدارة ملفها الطبي، استقبال الحجوزات والرد على المرضى.

## المزايا

- تسجيل دخول/حساب موحّد لنوعين من المستخدمين: **مريض** و **منشأة طبية (عيادة/مستشفى)**
- بحث وفلترة العيادات حسب المدينة، التخصص، ونوع المنشأة
- حجز موعد مباشرة من صفحة العيادة
- لوحة تحكم للمريض: متابعة الحجوزات والمحادثات
- لوحة تحكم للمنشأة: تأكيد/إلغاء الحجوزات، تعديل بيانات المنشأة، الرد على المرضى
- دردشة فورية (Realtime) بين المريض والمنشأة عبر Firestore
- مرتبط مباشرة بمشروعك على Firebase (Authentication + Firestore)

## التقنيات

React 19 + Vite + React Router + Tailwind CSS 4 + Firebase (Auth, Firestore).

## 1) التشغيل محلياً

```bash
npm install
npm run dev
```

سيعمل الموقع على `http://localhost:5173`.

للبناء النهائي (ملفات جاهزة للنشر):

```bash
npm run build
```

ستجد الناتج داخل مجلد `dist/`.

## 2) إعداد مشروع Firebase (مهم قبل التشغيل)

بيانات مشروعك موجودة بالفعل في `src/firebase.js`. تبقى عليك هذه الخطوات من
[Firebase Console](https://console.firebase.google.com/):

### أ) تفعيل تسجيل الدخول بالبريد وكلمة المرور
Authentication ← Sign-in method ← فعّل **Email/Password**.

### ب) إنشاء قاعدة بيانات Firestore
Firestore Database ← Create database ← اختر وضع **Production mode** (سنطبّق
قواعد الحماية يدوياً في الخطوة التالية) ← اختر المنطقة الأقرب لك.

### ج) تطبيق قواعد الحماية (Security Rules)
في تبويب **Rules** داخل Firestore، الصق محتوى ملف `firestore.rules` الموجود
في جذر هذا المشروع، ثم اضغط **Publish**.

تضمن هذه القواعد أن:
- أي زائر يمكنه تصفح العيادات بدون تسجيل دخول
- كل مستخدم يصل فقط لحجوزاته ومحادثاته الخاصة
- فقط صاحب المنشأة يمكنه تعديل بياناتها أو تأكيد حجوزاتها

### د) (اختياري) فهارس Firestore
التطبيق مصمم لتفادي الحاجة لفهارس مركّبة (composite indexes) عبر الفرز في
الواجهة الأمامية. إذا ظهرت لك رسالة خطأ في الـ Console تطلب إنشاء index، اتبع
الرابط الذي تعرضه Firebase مباشرة وسينشئه تلقائياً.

## 3) هيكل البيانات في Firestore

| المجموعة (Collection) | الوصف |
|---|---|
| `users/{uid}` | بيانات المستخدم: الاسم، البريد، الهاتف، والدور (`patient` أو `provider`) |
| `clinics/{uid}` | بيانات المنشأة الطبية (نفس uid لصاحب الحساب): الاسم، النوع، التخصص، المدينة... |
| `appointments/{id}` | طلبات الحجز، تتضمن `patientId` و `clinicId` والحالة |
| `chats/{patientId_clinicId}` | محادثة بين مريض ومنشأة |
| `chats/{chatId}/messages/{id}` | رسائل المحادثة (فورية عبر `onSnapshot`) |

## 4) النشر (Deployment)

### عبر Firebase Hosting

```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # اختر مشروعك sudan tec-5fee8، ومجلد public = dist
npm run build
firebase deploy
```

### أو عبر Vercel / Netlify
ارفع المشروع كما هو (يكتشف Vite تلقائياً)، أو اسحب مجلد `dist/` بعد `npm run build`.
